第二章 Linux基础篇

Linux入门

Linux介绍

  • Linux时一款操作系统，免费，开源，安全，高效，稳定（服务器运行半年不需要重新启动，而Windows却不稳定），处理高并发非常强悍。企业级项目大多都部署在Linux上。.net一般会在Windows平台
  • 创始人：linux（林纳斯），吉祥物：企鹅（tux，晚礼服）。
  • linux内核是核心，各种发行版是在内核上添加各种应用软件。
  • Linux（菜）主要发行版（西红柿炒蛋）：
    • Centos（前身是Redhat）
    • Redhat
    • Ubuntu
    • Suse
    • 红旗Linux
  • 目前主要的操作系统：Windows，android，Mac，IOS。

Linux和Unix关系

  • Ken tompson :C 语言和B语言创始人

  • Dennis richres：C语言之父

  • Richard Stallman：源码应该供用户免费开放

    • GNU计划
      • 人=>软件=>shell=>内核=>硬件
      • 其中内核的开发引导着Linux的诞生

  • 贝尔实验室：

    • 通讯卫星
    • 有声电影
    • 晶体管

  • Unix一开始用B语言做的，贝尔实验室73年用C语言实现

  • Unix是开源的

  • 由AT&T System V版本演变来的Minix

    • 林纳斯在Minix的基础上开发出了Linux内核，最后一只演变到现在

Linux和Windows比较

  • windows
    • 贵
    • 安全性弱
    • 图形界面方便
    • 可定制性差
    • 桌面应用
  • Linux
    • 开源，自由软件
    • 相比Windows安全得多（原因：开源，每次发现漏洞就会发布，维护次数多）
    • 命令行操作强大
    • 开源，可定制
    • 服务器应用

Windows，VM 和 CentOS的关系

  • Windows：本机
  • VM软件：虚拟机空间
  • CentOS:一个系统在虚拟机空间运行
  • BIOS里修改设置开启虚拟化设备支持

CentOS安装

网络设置

  • 桥连接模式：虚拟机的IP和主机的IP在一个网段
    • 可以和其他的系统通信
    • 由于在一个网段，可能造成IP冲突
  • NAT模式
    • 在主机上除了自己本身的IP网段，还会多出来一个IP网段，多出来的IP网段是Linux可以使用的IP网段
    • 别的主机访问不到Linux，但是Linux可以通过主机访问到外网
  • 主机模式
    • Linux是一个独立的主机，不能访问外网
  • 一般情况下选择NAT模式

正式安装

  1. 选择ISO镜像文件
  2. next
  3. 在做开发时为主机设置根密码时，绝对不允许使用123456这种简单密码，很容易被破解。去自动生成随机密码。
  4. 手动分区
    • boot分区：引导文件（200M）
    • swap分区：交换分区，暂时替代的虚拟分区（2048M（分配物理内存的1-1.5倍））
    • 根分区L ：18231M
  5. 自定义安装桌面（尽量小）
    • 兼容程序库
    • 调试工具
    • 互联网浏览器
    • 桌面保留
    • 语言中文
  6. Kdump
    • 内核崩溃转储机制。

文件系统目录结构

  • 只有一个根目录“/”，是级层式的树结构

  • Linux世界里，一切皆文件

  • /dev 管理设备：里面有cpu，disk等

  • /media 媒体目录 u盘，光驱

  • /bin：最经常使用的命令

  • /sbin：系统管理员的管理程序

  • /home：普通用户的主目录

  • /lib：动态链接共享库

  • /etc：配置文件和子目录

  • /usr：用户安装的软件

  • /boot：引导系统

  • /mnt：临时挂载别的文件系统

  • /opt：安装软件所排放的目录

  • /usr/local：安装过后的目录

  • /var：日志文件和经常修改的文件

开机，重启和用户注销

开机重启命令

  • shutdown
    • shutdown -h now：立即关机
    • shutdown -h 1：1分钟后关机
    • shutdown -r now：立即重启
  • halt：关机命令
  • reboot：重启
  • sync：将内存的数据同步到磁盘上，在关机和重启之前最好使用sync

用户登录和注销

  • 尽量少使用root账号登陆，权限为最大，避免操作失误造成系统崩溃。一般以普通用户登录，然后用su登陆root
  • logout注销账户，在远程登陆（运行级别3）上使用，在图形界面是无效的。

用户管理

  • Linux具有用户组
    • root组
    • 其他组
  • Linux具有用户家目录
    • 根目录下的home目录下有各个用户对应的家目录，用户登录时会自动进入自己的家目录

添加用户

  • 使用者要想使用系统必须要申请一个用户

  • useradd [可选] 用户名（root用户下执行）

    • 执行完后会在home目录下生成一个设置的用户名的家目录
    • 也可以指定目录进行创建useradd -d /home/zx zx1

修改密码

  • 修改密码passwd zx1

删除用户

  • 保留家目录
    • userdel zx1
  • 同时删除家目录
    • userdel -r zx1
  • 实际使用时都会保留家目录

查询用户信息

  • 如果用户不存在，返回无此用户

  • id zx1

切换用户

  • su zx1

    • 切换到zx1用户

  • 高权限用户到低权限用户不需要密码

  • exit

    • 回到原来的用户，回到高等级用户不需要密码

用户组

  • 一个用户创建后会归属到一个用户组
  • 系统对有共性的用户进行统一的管理

增加用户组

  • groupadd zx

删除用户组

  • groupdel zx

增加用户时指定用户组

  • useradd -g zx zx1

修改用户组

  • usermod -g zz zx1
    • 将用户zx1修改到zz组
